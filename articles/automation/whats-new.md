# What's new in Azure Automation?

Azure Automation receives improvements on an ongoing basis. To stay up to date with the most recent developments, this article provides you with information about:

- The latest releases
- Known issues
- Bug fixes

This page is updated monthly, so revisit it regularly.

## January 2020 to December 2020

The following is a rollup of all the changes performed on Azure Automation and its features over the current year. Included are new features, changed features, and deprecated functionality.

- Support for Impact Level 5 (IL5) compute isolation in Azure Government cloud.
- Updated runbooks used by Start/Stop VMs during off-hours to use Azure Az modules
- Retirement of the Automation watcher task
- Support for Update Management feature in the following regions:

   - South Central US
   - Switzerland North
   - China East 2
   - US Gov Arizona

- Support for Azure Az module cmdlets in runbooks
- Automation account availability in the following US Government cloud regions:

   - US Gov Virginia
   - US Gov Arizona

- Introduced public preview support of customer-managed keys stored in Azure Key Vault.
- Introduced public preview support of Private Link to set up a private endpoint with your Automation account.
- Updated Automation service DNS records from region-specific to Automation account-specific URLs.
- Introduced support for Azure virtual network service tags.
- Enforcement of TLS 1.2 and later versions with Automation service.
- Windows DSC extension version update.
- Private preview of Python3 support in West Central US and Japan East regions.
- Support for Automation account and State Configuration in the following regions:

   - Brazil South East
   - Norway East
   - Switzerland North

- Automation diagnostic logs schema update.
- Hybrid Runbook Worker support for Windows Server 2008 R2.
- Retired support of Azure Classic PowerShell cmdlets.
- Introduced native support for Azure Lighthouse in Update Management.
