# Azure Deployment Environments - maintenance mode

[!INCLUDE [note-deployment-environments-maintenance-mode](includes/note-deployment-environments-maintenance-mode.md)]

This article explains what maintenance mode means for Azure Deployment Environments (ADE) and how to plan for existing usage.

## What maintenance mode means

- Existing ADE capabilities remain available and supported for current usage.
- No new ADE features are planned.
- Existing documentation remains the best source for setup, operations, and troubleshooting guidance.

## Planning guidance for existing deployments

If you're currently using ADE, continue to operate with your existing governance and lifecycle practices.

Use these articles for ongoing operations:

- [Configure Azure Deployment Environments](quickstart-create-and-configure-devcenter.md)
- [Create and access an environment](quickstart-create-access-environments.md)
- [Create and access an environment by using the Azure CLI](how-to-create-access-environments.md)
- [Troubleshoot custom image errors and warnings](troubleshoot-custom-image-logs-errors.md)
