Azure Storage Mover supports both agent-based and agentless migration workloads. For agent-based workloads, a migration agent VM runs in your environment near the source storage. For agentless workloads, no migration agent VM is required.

Migration orchestration and management are provided by the cloud service for both workload types. For agent-based workloads, details about deployment and registration are available in the [Storage Mover agent deployment](../agent-deploy.md) and [agent registration](../agent-register.md) articles.
